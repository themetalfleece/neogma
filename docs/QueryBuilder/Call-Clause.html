linkCall (Subquery)

QueryBuilderParameters['CallI']

The call method allows you to create CALL subqueries in your Cypher query. CALL subqueries are useful for operations that need isolated scope, post-UNION processing, aggregations within OPTIONAL MATCH, and eager loading patterns.

linkCall with a literal string

A literal string will be wrapped in a CALL block.

⚠️ Security Warning: When using a literal string, the content is inserted directly into the query without sanitization. Never pass user-provided input directly to this method. Use BindParam for parameterized values and validated/escaped identifiers for dynamic elements. Prefer using QueryBuilder instances (shown below) which provide safer parameter binding.

1linkconst queryBuilder = new QueryBuilder()

2link    .match({ identifier: 'n', label: 'Node' })

3link    .call(`

4link        WITH n

5link        MATCH (n)-[:RELATES_TO]->(m:Related)

6link        RETURN collect(m) AS related

7link    `)

8link    .return(['n', 'related']);

9link

10linkconsole.log(queryBuilder.getStatement());

1linkMATCH (n:Node) CALL { WITH n MATCH (n)-[:RELATES_TO]->(m:Related) RETURN collect(m) AS related } RETURN n, related

Note: getStatement() collapses whitespace for a compact single-line output.

linkCall with a QueryBuilder instance

You can pass another QueryBuilder instance to the call method. The subquery's statement will be extracted and wrapped in a CALL block. Both QueryBuilder instances share the same BindParam for consistent parameter binding.

1linkconst bindParam = new BindParam();

2link

3link// Build the subquery

4linkconst subquery = new QueryBuilder(bindParam)

5link    .with('parent')

6link    .match({

7link        related: [

8link            { identifier: 'parent' },

9link            { direction: 'out', name: 'HAS_CHILD' },

10link            { identifier: 'child', label: 'Child' }

11link        ]

12link    })

13link    .return('collect(child) AS children');

14link

15link// Build the main query

16linkconst queryBuilder = new QueryBuilder(bindParam)

17link    .match({ identifier: 'parent', label: 'Parent' })

18link    .call(subquery)

19link    .return(['parent', 'children']);

20link

21linkconsole.log(queryBuilder.getStatement());

1linkMATCH (parent:Parent) CALL { WITH parent MATCH (parent)-[:HAS_CHILD]->(child:Child) RETURN collect(child) AS children } RETURN parent, children

linkNested Call subqueries

You can nest CALL subqueries by calling call within the subquery builder.

1linkconst bindParam = new BindParam();

2link

3link// Inner subquery

4linkconst innerSubquery = new QueryBuilder(bindParam)

5link    .with('child')

6link    .match({

7link        related: [

8link            { identifier: 'child' },

9link            { direction: 'out', name: 'HAS_ITEM' },

10link            { identifier: 'item', label: 'Item' }

11link        ]

12link    })

13link    .return('collect(item) AS items');

14link

15link// Outer subquery

16linkconst outerSubquery = new QueryBuilder(bindParam)

17link    .with('parent')

18link    .match({

19link        related: [

20link            { identifier: 'parent' },

21link            { direction: 'out', name: 'HAS_CHILD' },

22link            { identifier: 'child', label: 'Child' }

23link        ]

24link    })

25link    .call(innerSubquery)

26link    .return('collect({ node: child, items: items }) AS children');

27link

28link// Main query

29linkconst queryBuilder = new QueryBuilder(bindParam)

30link    .match({ identifier: 'parent', label: 'Parent' })

31link    .call(outerSubquery)

32link    .return(['parent', 'children']);

33link

34linkconsole.log(queryBuilder.getStatement());

1linkMATCH (parent:Parent) CALL { WITH parent MATCH (parent)-[:HAS_CHILD]->(child:Child) CALL { WITH child MATCH (child)-[:HAS_ITEM]->(item:Item) RETURN collect(item) AS items } RETURN collect({ node: child, items: items }) AS children } RETURN parent, children

linkUse Cases

linkEager Loading Relationships

CALL subqueries are ideal for loading related nodes with their relationships in a single query, avoiding N+1 query problems.

1linkconst bindParam = new BindParam();

2link

3linkconst ordersSubquery = new QueryBuilder(bindParam)

4link    .with('user')

5link    .match({

6link        related: [

7link            { identifier: 'user' },

8link            { direction: 'out', name: 'PLACED', identifier: 'rel' },

9link            { identifier: 'order', label: 'Order' }

10link        ],

11link        optional: true

12link    })

13link    .orderBy({ identifier: 'order', property: 'createdAt', direction: 'DESC' })

14link    .limit(10)

15link    .return('collect({ node: order, relationship: rel }) AS Orders');

16link

17linkconst queryBuilder = new QueryBuilder(bindParam)

18link    .match({ identifier: 'user', label: 'User' })

19link    .where({ user: { status: 'active' } })

20link    .call(ordersSubquery)

21link    .return(['user', 'Orders']);

linkPost-UNION Processing

CALL subqueries can wrap UNION operations to apply operations on the combined results.

1linkconst queryBuilder = new QueryBuilder()

2link    .call(`

3link        MATCH (n:TypeA) RETURN n.name AS name

4link        UNION

5link        MATCH (n:TypeB) RETURN n.name AS name

6link    `)

7link    .return('name')

8link    .orderBy('name');
