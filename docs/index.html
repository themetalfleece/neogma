Object-Graph-Mapping neo4j framework, fully-typed with TypeScript, for easy and flexible node and relationship operations

linkOverview

Neogma uses Model definitions to simplify and automate lots of operations. Alternatively, a flexible and fully-fletched query builder and a query runner is also provided for running operations directly with Javascript objects, without a Model definition.

By using Typescript, a user can also benefit from Neogma's type safety in all its parts. The types are built-in and used in neogma's core, so no external typings are needed.

linkExamples

You can try the following to see it in action!

First install neogma: npm i neogma

Then, run this:

1const { Neogma, ModelFactory } = require('neogma');

2

3/* --> create a neogma instance and database connection */

4const neogma = new Neogma(

5    {

6        /* --> use your connection details */

7        url: 'bolt://localhost',

8        username: 'neo4j',

9        password: 'password',

10    },

11    {

12        logger: console.log,

13    }

14);

15

16/* --> create a Users model */

17const Users = ModelFactory({

18    label: 'User',

19    schema: {

20        name: {

21            type: 'string',

22            minLength: 3,

23            required: true

24        },

25        age: {

26            type: 'number',

27            minimum: 0,

28        },

29        id: {

30            type: 'string',

31            required: true,

32        }

33    },

34    primaryKeyField: 'id',

35    relationshipCreationKeys: {},

36}, neogma);

37

38const createAndUpdateUser = async () => {

39    /* --> creates a new Users node */

40    const user = await Users.createOne({

41        id: '1',

42        name: 'John',

43        age: 38

44    });

45

46    console.log(user.name); // 'John'

47

48    user.name = 'Alex';

49    /* --> updates the node's name in the database */

50    await user.save();

51    console.log(user.name); // 'Alex'

52

53    await neogma.getDriver().close();

54}

55

56createAndUpdateUser();

The Cypher which runs in createAndUpdateUser is the following:

1Statement: UNWIND {bulkCreateOptions} as bulkCreateData CREATE (bulkCreateNodes:`User`) SET bulkCreateNodes += bulkCreateData

2Parameters: { bulkCreateOptions: [ { name: 'John', age: 38, id: '1' } ] }

3

4Statement: MATCH (node:`User`) WHERE node.id = $id SET node.name = $name

5Parameters: { id: '1', name: 'Jack' }

Another feature is to associate the created nodes with other nodes, which will either be created now or by matched by a where clause. This supports infinite nesting for maximum flexibility.

1/* --> create a User node, an Order node and a relationship between them */

2await Users.createMany([

3    {

4        id: '1',

5        name: 'John',

6        age: 38,

7        /* --> assuming we're created an Orders Model and alias */

8        Orders: {

9            properties: [{

10                /* --> creates a new Order node with the following properties, and associates it with John */

11                id: '1',

12                status: 'confirmed'

13            }],

14            where: {

15                params: {

16                    /* --> matches an Order node with the following id and associates it with John */

17                    id: '2'

18                }

19            }

20        }

21    }

22]);

23

24// find the Order node which is created in the above operation

25const order = await Orders.findOne({

26    where: {

27        id: '1'

28    }

29});

30

31console.log(order.status); // confirmed

The cypher which runs in Users.createMany is the following:

1Statement: CREATE (node:`User`) SET node += $data CREATE (node__aaaa:`Order`) SET node__aaaa += $data__aaaa CREATE (node)-[:CREATES]->(node__aaaa) WITH DISTINCT node MATCH (targetNode:`Order`) WHERE targetNode.id = $id CREATE (node)-[r:CREATES]->(targetNode)

2

3Parameters: {

4  data: { name: 'John', age: 38, id: '1' },

5  data__aaaa: { id: '1', status: 'confirmed' },

6  id: '2'

7}

All of the above run in a single statement for max performance. The association alias (here Orders) is defined in the Model along with the relationship name, direction and properties.

All the user-specified values are automatically used in the query with bind parameters. Neogma also offers helpers to easily create your own queries with generated where clauses and bind parameters.
